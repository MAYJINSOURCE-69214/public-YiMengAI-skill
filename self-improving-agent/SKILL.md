## 🔍 日志规范自检引擎（10 大维度 + 14 天扫描）

**核心逻辑**：按 `docs/log-format.md` v26.1.0 标准，逐项扫描日/周/月日志文件，量化评分，自动修复。

### 日日志检查（10 大维度 v30.0.0 升级）

| 检查维度 | 检查内容 | 修复动作 |
|----------|----------|----------|
| **1. 标题格式** | 是否为 `# YYYY-MM-DD 工作日志` | 自动修正标题行 |
| **2. 核心主题** | 是否有 `## 📖 今日核心主题` + `>` 引用块 | 缺失则补空区块 |
| **3. 5 个时段标题** | 是否为标准 5 时段（凌晨/早上/中午/下午/晚上） | 修正自定义时间段标题 |
| **4. 汇报格式** | 是否为 `### 09:00 早安汇报` 等标准格式 | 修正汇报标题 |
| **5. 分隔线** | 汇报与事项间是否有 `▬▬▬▬▬▬▬▬▬▬▬▬▬▬▬▬▬▬▬▬▬▬▬▬▬▬▬▬▬▬▬▬▬` | 缺失则插入 |
| **6. 13 个 L3 区块** | 是否有核心 7 个 + 回忆录 6 个（学习/情绪/社交/健康/财务/感恩） | 缺失则补标准框架 |
| **7. 事项总结格式** | 是否用 `事项名称（时间段）` + `内容细节：` 格式，**严禁用 `###` 分割** | 修正为无 `###` 格式 |
| **8. 空模块默认话术** | 无内容时是否填写默认话术（如 `今日无新学习收获`），**严禁省略区块** | 补默认话术 |
| **9. 文件名格式** | 是否为 `YYYY-MM-DD.md`（如 `2026-05-20.md`），**严禁短格式 `DD.md`** | 自动重命名 |
| **10. 凌晨归属** | 00:00-05:59 的活动是否归属到**第二天**的凌晨时段 | 修正归属错误 |

### 周/月日志检查

| 日志类型 | 检查内容 | 修复动作 |
|----------|----------|----------|
| **周日志** | 文件名是否为 `YYYY-WXX.md`（如 2026-W20.md） | 重命名旧格式文件 |
| **月日志** | 文件名是否为 `YYYY/MM.md`（如 2026/05.md） | 修正路径 |
| **核心主题** | 是否有本周/月核心总结 | 缺失则提示补充 |
| **大事记** | 是否有完整大事记时间线 | 缺失则从日日志提炼 |
| **13 个 L3 区块** | 是否有核心 7 个 + 回忆录 6 个（月以上可合并） | 缺失则补标准框架 |

### 自检流程（v30.0.0 增强版）

1. **扫描**：读取近 14 天日日志文件 + 近 4 周周日志 + 近 2 月月日志
2. **逐项检查**：对每个文件执行 10 大维度检查（日）+ 周/月格式检查
3. **生成报告**：输出合规率表格（文件/合规项/总项/百分比）+ 错误趋势分析
4. **问题清单**：列出不规范问题（文件路径 + 偏离描述 + 风险等级）
5. **自动修复**：直接执行批量修复（edit/write_file），无需主人确认
6. **记录错误**：写入 `memory/learnings/errors.md` + 更新错误趋势统计
7. **记忆完整性补充**：发现缺失日志层级时，自动从下往上总结提炼补齐
